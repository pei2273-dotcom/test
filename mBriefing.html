✈️ LIDO mBriefing 電子飛行計劃操作流程

以下僅為航機簽派部“建議的” SOP, 仍依單位需求為主

FDP 報到
📱
登入與航班確認
使用帳號密碼或 Face ID 登入 LIDO mBriefing APP
確認是否已有航班資訊（系統會依組員名單自動顯示）
⚠️ 航班資訊處理流程 ▶
❌ 沒有航班資訊 → 點選 Search，輸入航班號碼 (如：0100) 進行搜尋
❌ 搜尋不到航班 → 繼續通勤，進入駕駛艙後再下載計畫
✅ 有航班資訊 → 直接下載計畫
依組員需求通知 OD 修改計畫
或透過 Briefing - Fuel - Order 功能申請額外油量
ℹ️ Fuel Order 功能說明 ▶
功能：可輸入加油原因與數值，並自動連動至 L/S、Fuel Sheet，並可透過 ACARS 發送訊息給 OD/MM/LD
📖 是否會收到完整新計畫 ▶
以 FOM 4.8.3 Short Plan 為準
如檢視後不需修改 → 由 Captain 點選 Accept Flight
⚠️ 僅 Captain 有權限執行 Accept Flight
✅ Accept Flight 功能說明 ▶
功能：確認已接受電子飛行計畫
Re-accept：如開啟後發現已被人接受，可點選右上角功能，Re-accept Flight 再次確認
追蹤功能：後台可以追蹤是誰誤 Accept Flight
進入駕駛艙
✈️
駕駛艙檢查與確認
確認航班能下載開啟
由 Captain 點選 Accept Flight（確認已接收電子飛行計畫）
❌ 無法開啟檔案處理 ▶
如仍無法開啟檔案 → 啟用 "紙本計畫" 流程
✅ Accept Flight 功能說明 ▶
功能：確認已接受電子飛行計畫
Re-accept：如開啟後發現已被人接受，可點選右上角功能，Re-accept Flight 再次確認
追蹤功能：後台可以追蹤是誰誤 Accept Flight
網路與同步確認
📡
網路與同步確認
組員啟用 Real Time Sync 或備援 Share
完成飛行計畫檢查並確認無誤
ℹ️ Real Time Sync 功能說明 ▶
功能：App 內建同步功能，除 Scratchpad 外，其他資訊皆會同步
需求：需開啟藍芽
限制：同時間，只能有一人開啟 Real Time Sync 房間，不能多人同時開啟
紀錄邏輯：依最新的時間判斷
ℹ️ Share 功能說明 ▶
功能：由組員點選需要 Share 資訊給指定的人
計畫確認
📋
ACARS Accept 確認
依據計畫編號，由 PIC 透過 ACARS Accept 確認計畫
📖 相關法規 ▶
AOR 第54條-1 七 與 第55條
計畫內容 (NOTAM、ATC、General) 之修改需求，由航務處定義
⚠️ 簽派部建議討論 ▶
是否保留標籤頁面 ETP information，請參考附圖討論
特殊狀況處理
🔄
GTB / ATB / 特殊狀況
❌ 返回登機門處理 ▶
若返回登機門並收到新計畫 → 需重新回到 Accept Flight 流程
（確認接收到電子飛行計畫，僅 Captain 可執行）

組員再次透過 Real Time Sync 確認，同步完成並且檢查完資料後，使用 ACARS Accept Release 計畫
巡航階段
🛫
巡航中 LIDO mBriefing App 異常
須再確認全員是否於同一 Sync 房間
❌ Sync 房間失敗處理 ▶
如失敗 → 全員退出，再由 Captain 建立新房間
❌ App 異常處理 ▶
若 App 發生異常 → 透過 ACARS 由簽派部傳送飛行計畫
落地後
📤
Submit Flight 作業
PIC 檢查資料無誤，確認 POST FLIGHT DATA 已上傳至 Package
點選 Submit Flight（代表：送出"個人"飛行資料）
✅ Submit Flight 功能說明 ▶
功能：上傳"個人"飛行資料
查看：上傳後可由 Flight list -> submitted 看到送出的航班與時間
修改：如有需要更改，仍可進去航班內，修改後再次 Submit
後台設定：抓取 PIC 最新上傳的資料
ℹ️ 飛行後平台設定 ▶
飛行後平台，可以設定只顯示 PIC 的文件
文件管理
📁
POST Flight Data 與文件處理
📖 Master Flight Plan 討論建議 ▶
相關條文：FOM 4.8.2
因電子飛行計畫可全部上傳至 Server，並具備 Real Time Sync 功能
簽派建議：所有組員皆可 Submit，後台僅保留 PIC（由 ACARS Accept 指定）的資料，以符合 Master 規範
⚠️ TO DATA 處理建議 ▶
方案 1：截圖上傳，檔名：TOW（大小寫皆可）
方案 2：填寫於 Notes 頁面
方案 3：填寫於 Additional Briefing Documents
ℹ️ 其他文件處理方式 ▶
NOTOC：依據 FOM6.2.4 與 GOM5.7.2，公司已在 Loading 與 CGO 留存文件副本，組員值勤結束後建議妥善丟棄 NOTOC

LoadSheet：依據 FOM4.9.5，公司已透過 ACARS 電子簽章簽名
若為 manual 文件 → 可拍照上傳，檔名：LS（大小寫皆可）
如影像模糊 → 以 Loading 保存之副本為主
⚠️ Operation Log 方案討論 ▶
方案一：
• 客製化 LIDO mBriefing -- Journey Log 用以符合公司 Operation Log 與 Rest Time 記錄規範
• 組員需同時完成 ACARS Operation Log 與 LIDO mBriefing Data
• 組員 Submit 後，後台僅擷取 PIC 的資料
• 如資料不足 → 再由 ACARS 補齊並要求組員補簽名
• 需獲取 AC120-78B 認證

方案二：
• 原 ACARS 資料照相上傳，檔名：LOG（大小寫皆可）
